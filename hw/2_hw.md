## Домашнее задание 2: Логика конвертера на `useState` и мок-данных

### Цель

Довести UI из первой домашки до рабочего состояния без подключения к API: вся логика конвертации, выбор валют и описание пары должны работать только на локальных данных.

### Что нужно реализовать

1. Подключить локальные мок-данные в проект как константы (например, в `src/mocks`):
    - `currencies: Currency[]`;
    - `priceChanges: Record<string, Record<string, PriceChange>>`.
2. Реализовать состояние конвертера через `useState`:
    - `from`, `to`, `amount`, `result`;
    - состояние видимости блока `More about ...`.
3. Сделать пересчет результата на локальных данных:
    - при изменении суммы;
    - при изменении пары валют;
    - курс для пары брать из `priceChanges[from][to]`.
4. Не допускать некорректную пару:
    - нельзя выбрать одинаковые валюты в обоих селектах;
    - при совпадении автоматически выбирать первую доступную отличающуюся валюту.
5. Добавить кнопку `Swap` (поменять `from` и `to` местами) с корректным пересчетом результата.
6. Обновить блок `More about ...`:
    - заголовок формируется из текущей пары (`More about RUB/USD`);
    - контент берется из `description` выбранных валют;
    - если описания нет, показывать fallback-текст.
7. Реализовать reset по `key`:
    - при смене валютной пары сбрасывать локальное состояние дочернего компонента описания (`open/closed`) через изменение его `key`;
    - коротко пояснить в коде, почему здесь выбран именно такой подход.

### Контракт данных

Во второй домашке локальные моки должны повторять будущий серверный контракт.

`Currency`:

- `code: string`
- `name: string`
- `description: string`
- `symbol: string`

`PriceChange`:

- `purchasedCurrencyCode: string`
- `paymentCurrencyCode: string`
- `price: number`
- `dateTime: string` (ISO 8601)

Требования к данным:

- завести TypeScript-типы для `Currency` и `PriceChange`;
- добавить в проект готовые моки из файлов `2_hw_mock_currencies.json` и `2_hw_mock_price_changes.json` как константы;
- использовать эти данные в UI конвертера.

Как работать с моками:

- `2_hw_mock_currencies.json` содержит список валют для обоих селектов;
- `2_hw_mock_price_changes.json` содержит курсы в структуре `fromCurrency -> toCurrency -> PriceChange`;
- курс для выбранной пары брать по пути `priceChanges[fromCurrency][toCurrency]`;
- значение `price` использовать для расчета результата конвертации;
- `dateTime` хранить как часть модели курса; в расчетах на этом этапе можно не использовать;
- `description` валют использовать в блоке `More about ...`.

### Тесты

Минимальный набор тестов:

- рендер селектов и полей с мок-данными;
- пересчет конвертации при изменении суммы/пары;
- запрет на одинаковые валюты в паре;
- reset дочернего состояния по `key`.

### Структура папок

Важно соблюдать понятную структуру проекта. Минимально ожидается разбиение по зонам ответственности:

- `src/models` - клиентские модели данных, с которыми работает UI;
- `src/hooks` - хуки;
- `src/components` - React-компоненты;
- `src/pages` - тут должна храниться страница `Main` с композицией ваших компонентов.

### Критерии приемки

- Проект запускается через `npm run dev`.
- Проект корректно билдится через `npm run build`.
- Тесты проходят через `npm run test`.
- Конвертация корректно работает на локальных данных для разных валютных пар.
- `Swap`, `More about ...` и reset по `key` работают стабильно.
